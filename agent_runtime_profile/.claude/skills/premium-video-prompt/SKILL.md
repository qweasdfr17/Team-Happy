# 精品视频提示词生成器

本 skill 是 ArcReel 的正式视频提示词生成器。提示词写作风格使用用户自定义方法论；项目读取、资产引用、前端红框写回仍遵守 ArcReel 规则。

## 最高优先级：只写提示词，不生成视频

- 只生成或更新 `video_prompt` / reference_video unit prompt。
- 不生成视频。
- 不调用 `generate_video_*` 系列工具。
- 写完后回复用户：提示词已写入，请在页面审核，确认后再生成视频。

## ArcReel 适配规则

生成提示词前必须读取：

- `project.json`：读取 `style`、`style_description`、`aspect_ratio`、characters、scenes、props、products。
- 当前集 `scripts/episode_N.json`：读取当前 scenes / segments / video_units 的完整原文、对话/旁白、已有 image_prompt / video_prompt。
- context_pack：只作为辅助摘要，不能替代原文读取。
- prompt_library：可作为参考，但不得覆盖本 skill 的输出结构和写作方法论。

写回规则：

- reference_video 模式：调用 `mcp__arcreel__patch_reference_video_unit_prompt` 写入当前 unit prompt。
- storyboard/grid 模式：调用 `mcp__arcreel__patch_episode_script` 写入对应分镜的 `video_prompt`。
- reference_video 写回时必须传递 `references` 参数，格式为 `[{type, name}]`，顺序和提示词里的资产引用顺序一致。
- 禁止 Bash/Write/Edit 直接修改 `scripts/*.json` 或 `project.json`。

## script_policy 约束

- `preserve`：只能基于原文生成视觉补充，不得改写原文剧情和对白含义。
- `suggest_rewrite`：如需改稿建议，写入 `proposals/`，不覆盖正式剧本。
- 生成的提示词是派生内容，不得覆盖原始剧本文字。

## 资产引用规则

### reference_video 模式

- 提示词开头必须使用项目资产引用语法：`@资产名` 或 `@[复杂资产名]`。
- 资产名必须来自 `project.json` 已注册的 characters / scenes / props / products。
- 不要使用“图片1：资产名”或“[图1] 资产名”格式。
- 不要只写资产名而不加 `@`。
- 不要在资产引用里写角色外貌、服装、场景细节、括号说明之外的描述。
- 角色/场景/道具/产品的细节写到【基础设定】或【场景光影基调】中。
- 画面内容时间轴里只写角色名或自然称呼，不反复写 `@资产名`。

### storyboard/grid 模式

- 使用现有分镜图作为起始帧，不要在提示词里虚构不存在的参考资产。
- 角色、场景、道具名称必须和项目资产一致。
- `video_prompt` 重点写动作演变、镜头运动、台词/音效时间点，不改变 `image_prompt` 已定义的画面前提。

## 默认输出结构

每个分镜或 video unit 独立写完整结构，不共享、不省略：

```text
【分镜X | 时长 Xs】

【资产引用】（reference_video 模式必写；storyboard/grid 可省略）
@角色A，@场景B，@道具C

【基础设定】
角色只写“角色名@资产引用名”格式；场景只写时空和关键信息。

【画风】
根据 project.json 的 style / style_description 改写。若项目未设置明确风格，默认使用真人短剧质感：4K 数字拍摄，画面清晰锐利，肤色自然健康，真实皮肤纹理，明亮通透日光线，柔光补面避免脸部死黑，色彩饱和度适中略带鲜亮，浅景深突出主体，背景虚化柔和，无胶片颗粒，无暗角。

【场景光影基调】
写清楚光源方向、光质、阴影特征、空气介质和色调倾向。

不要背景音乐，不要字幕，仅保留音效和环境音。

【画面内容】
按秒拆解：景别、运镜、焦距、构图、动作、台词、音效。每个分镜从 0s 重新计秒。

【负面约束】
列出本镜禁止项。
```

## 写作核心原则

把“感觉”翻译成“参数”：

- 不写“紧张氛围”，写“浅景深 + 前推镜头 + 呼吸声渐重 + 冷蓝侧光 + 阴影压住半张脸”。
- 不写“她很害怕”，写“第 2 秒瞳孔收缩，第 3 秒肩膀后缩，第 4 秒嘴唇发抖，第 5 秒屏住呼吸”。
- 每个抽象词都拆成具体的视觉、听觉、时间参数。

## 台词和时长规则

对话比画面更占时长。写提示词前先估算台词时间：

| 台词长度 | 参考耗时 |
| --- | --- |
| 1-5 字短句 | 约 1 秒 |
| 5-10 字中句 | 约 2 秒 |
| 10-20 字长句 | 约 3-4 秒 |
| 20 字以上 | 约 5 秒以上 |

经验公式：

```text
台词总时间 + 动作/反应时间（每镜约 3-5 秒） + 留白呼吸时间（每镜约 1-2 秒） = 分镜总时长
```

必须读取上游分镜的“对话/旁白”。如果有对白或旁白，必须写进【画面内容】时间轴里，标明说话人和台词内容；不能只写动作和音效。写完后按正常语速读一遍，读不完就加时长或拆分。

## 秒级画面内容规则

每条画面内容必须包含：

- 时间范围，例如 `0s~2s`
- 景别
- 运镜
- 焦距
- 构图
- 动作描述
- 台词时间点
- 音效时间点

动作要有先后因果关系，写“先 A → 然后 B → 最后 C”，不要只写结果。

示例：

```text
0s~2s：中近景，50mm 标准焦段，斯坦尼康平稳前推。欧阳韬伸手，先抓住桌沿稳住身体，然后俯身拉冉乐乐起身。第 1 秒衣料摩擦声，第 2 秒冉乐乐短促吸气。
2s~5s：近景，85mm 中长焦压缩背景。冉乐乐抬眼看他，眼眶微红，嘴角下压，肩膀轻颤。冉乐乐低声说：“我没有退路了。”
```

## 景别与焦距

| 景别 | 用法 |
| --- | --- |
| 大远景 | 建立时空、史诗感、人物渺小 |
| 远景/全景 | 交代人物与环境关系、动作轨迹 |
| 中景 | 叙事主力，兼顾表情和环境 |
| 中近景 | 强调表情和肩部语言 |
| 近景 | 压迫观众注视微表情 |
| 大特写 | 聚焦眼泪、手、钥匙、手机等关键线索 |
| 过肩镜头 | 对话权力关系和主观位置 |
| 双人镜头 | 展示亲密、对峙、距离变化 |
| 插入镜头 | 放大关键叙事元素 |

| 焦距 | 情绪效果 |
| --- | --- |
| 16-24mm 广角 | 压迫、混乱、不安 |
| 28-35mm 中广角 | 客观叙事、纪实、空间呼吸 |
| 40-50mm 标准 | 自然、中立、亲密 |
| 75-85mm 中长焦 | 人像质感、专注、优雅 |
| 100-135mm 长焦 | 孤独、窥视、压迫、空间割裂 |

## 运镜策略

默认使用斯坦尼康平稳跟拍，不加手持晃动，不加慢动作。只有特殊情绪需要时才改变。

选择运镜时先问：这个运动告诉观众什么？

- 紧张、压迫、对峙：前推、环绕、固定但压迫
- 释放、孤独、失落：后拉、固定 + 主体远离
- 混乱、真实、崩溃：手持、轻微不稳
- 沉浸、流畅、旁观：斯坦尼康跟拍、平移
- 窒息、失衡、震惊：推拉变焦、摇镜、倾斜
- 史诗、庄严、宏大：升降、航拍、固定大景

对话场景至少考虑三类覆盖：

1. 主镜头：全景/中景交代空间和关系
2. 过肩镜头：正反打建立对话位置
3. 单人中近景/近景：捕捉微表情

## 光影设计

光影描述顺序：

```text
光源方向 + 时间/色温 → 光质 → 阴影特征 → 空气介质 → 色调倾向
```

每镜必须写清楚：

1. 主光源在哪、什么方向、什么质感
2. 暗区在哪、为什么暗
3. 明暗交界带在哪
4. 是否有微尘、雾气、玻璃反光等空气介质
5. 光影如何服务人物情绪

可用打光风格：

- 伦勃朗光：侧上方主光，倒三角光斑，古典深沉
- 蝴蝶光：正前上方主光，优雅
- 侧光：一半亮一半暗，神秘、双重人格
- 逆光剪影：边缘轮廓，孤独、史诗、危险
- 顶光：眼窝变暗，压迫、恐怖
- 底光：反常阴影，诡异

## 情绪表演

情绪要按肌肉时间线拆：

- 眼眶 → 嘴角 → 鼻翼 → 肩膀 → 呼吸 → 声音 → 崩溃
- 加入呼吸变化：急促、屏住、深吸、啜泣
- 加入声音质感：哽咽、嘶哑、破音、气声

示例：

```text
0s~2s：眼眶微红，呼吸轻颤，指尖掐进掌心。
2s~4s：嘴角下撇，鼻翼翕动，泪水在眼眶打转。
4s~6s：单手捂嘴，肩膀耸动，发出压抑的呜咽。
6s~7s：放弃克制，泪水决堤，面部扭曲。
```

## 动作物理逻辑

写动作要写力的传导路径：

```text
脚掌蹬地 → 小腿发力 → 大腿肌肉绷紧 → 身体离地 → 空中滞停 → 手臂挥出
```

不要写“他打过去”，要写为什么能打过去、身体如何发力、对方如何反应。

## 音效设计

音效要和画面同步卡点：

- 标注发生秒数
- 标注质感：脆响、闷响、金属声、滚动声、衣料摩擦、呼吸声
- 多层叠加：环境底噪 + 具体事件音 + 人物呼吸/台词
- 不要背景音乐，不要字幕，仅保留音效和环境音

## 负面约束武器库

每个提示词最后必须写负面约束。按需选择：

- 质感：塑料感、过度磨皮、蜡像感、假人感、完美无瑕脸
- 运动：动作卡顿、机械动作、关节不自然
- 光影：光源方向不一致、面部光影突变
- 空间：人物与环境脱离、景深虚假、头发漂浮
- 后期：摩尔纹、噪点、画面闪烁、人物边缘闪烁、分辨率低
- 结构：角色变脸、人物瞬移、穿模、手指数量错误、新增人物、背景穿帮
- 风格：卡通化、AI 生成感、AI 视频感、塑料皮肤

## 输出质量检查

写回前自查：

- 是否使用本 skill 的默认输出结构
- 是否读取了当前分镜的对话/旁白
- 台词是否按秒安排，语速是否读得完
- 角色/场景/道具引用是否来自项目资产
- reference_video 是否传递 `references`
- 是否没有改变原文剧情
- 是否没有生成视频
- 是否没有写入不存在的资产引用
- 是否包含“不要背景音乐，不要字幕，仅保留音效和环境音”
- 是否包含负面约束
