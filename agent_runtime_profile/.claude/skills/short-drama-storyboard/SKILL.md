# Short Drama Storyboard

把长文本改成适合短视频平台传播的漫剧/短剧分镜脚本。

## ⚠️ script_policy 约束（最高优先级）

开始前先检查 `project.json` 的 `script_policy.mode`：

| 模式 | 行为 |
|------|------|
| **preserve**（默认） | 只能输出建议或派生分镜草案，**不能覆盖** scripts/episode_N.json。分镜提案写入 `proposals/` 目录 |
| **suggest_rewrite** | 生成分镜方案写入 `proposals/`，用户确认后通过受控工具写入正式项目 |

本 skill 是**创作型分镜 proposal**，不直接覆盖正式剧本。

## 适用范围

当用户提供以下任一内容时使用：
- 小说章节、网文片段
- 剧本初稿
- 故事梗概、人物设定
- 希望改成"短视频漫剧/短剧/即梦参考分镜"的文本

触发词：
- "把这个剧本改成分镜"
- "拆成短视频漫剧"
- "每集要有爆点和钩子"
- "先帮我改成适合短视频节奏"
- "做成可直接出片的分镜脚本"
- "先给我整体大纲，再生成完整分镜"

## 强制流程：必须分两步

**绝对不要一步到位直接输出完整分镜。**

### 第一步：先输出"整体更改大纲"

收到用户剧本后，先做内容审查和短视频化重构，只输出：

1. **剧本审查**
   - 核心卖点
   - 人设卖点
   - 当前不适合短视频平台的问题
   - 节奏/爆点/钩子/信息密度问题
   - 建议删、挪、提、压缩、前置的内容

2. **整剧拆集大纲**
   按每集 **1分30秒 - 2分30秒** 体量重构：
   - 每集标题
   - 开头爆点
   - 核心剧情
   - 结尾钩子

3. **改编说明**
   - 哪里重组了原文顺序
   - 哪些内容被强化/压缩
   - 哪些内容留到后集做钩子

然后**停下，等待用户确认。**

### 第二步：用户确认后，输出"完整分镜脚本"

只有用户明确确认大纲后，才生成完整分镜。

## 分镜输出规则

### 每个分镜的固定字段

1. **画面内容**
2. **即梦提示词**
3. **运镜信息**
4. **景别信息**
5. **对话信息**
6. **光影信息**
7. **摄像机 / 焦距 / 质感**
8. **声音设计**

### 分镜写法要求

- 一镜只承载一个核心信息点
- 每镜明确"这段画面要让观众看到什么"
- 开头几镜必须抓人：爆点、反差、冲突、悬念
- 每集形成：**开头爆点 → 中段推进 → 结尾钩子**
- 画面比例以 `project.json` 的 `aspect_ratio` 为准，不硬编码竖屏 9:16

## 平台适配原则

- 前3秒必须抓人
- 人物关系尽快清楚
- 冲突尽量前置
- 信息表达要直，不绕
- 不过长铺垫
- 每集结尾让人想点下一集

## 改编原则

允许适度改编，但保持：
- 主线不变
- 人设核心不变
- 关键情绪不变

可以做：
- 前置爆点、倒叙开场
- 压缩重复信息、合并场景
- 提前埋钩子
- 长段心理描写改成动作和冲突

不要做：
- 无理由魔改人物动机
- 复杂剧情改到逻辑断裂
- 跳过大纲确认直接生成全部分镜

## 输出风格

- 直接、清晰、制作导向
- 先结构，后分镜
- 重点始终放在"是否适合短视频出片"

## 第一步输出模板

```
### 一、剧本审查
- 核心卖点：
- 人设卖点：
- 当前问题：
- 改编建议：

### 二、整剧拆集大纲
#### 第1集｜标题
- 开头爆点：
- 核心剧情：
- 结尾钩子：

#### 第2集｜标题
- 开头爆点：
- 核心剧情：
- 结尾钩子：

### 三、改编说明
- 前置内容：
- 压缩内容：
- 留作钩子：
```

最后补一句：**"如果这版大纲你确认，我再按这版继续生成完整分镜脚本。"**

## 第二步输出模板

```
## 第X集｜标题

### 分镜1
- 画面内容：
- 即梦提示词：
- 运镜信息：
- 景别信息：
- 对话信息：
- 光影信息：
- 摄像机 / 焦距 / 质感：
- 声音设计：
```

## 关键提醒

- **先大纲，后分镜。**
- **用户未确认大纲前，不得直接生成完整分镜。**
- 分镜字段必须齐全，不要漏项。
- preserve 模式下只能输出到 `proposals/`，不覆盖正式剧本。
- 如用户确认要写入正式项目，走受控工具流程。
